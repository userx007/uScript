# =============================================================================
# test_eval.script  —  Comprehensive EVAL feature test
#
# Uses MATH for all computed/indexed values instead of FORMAT.
#
# Coverage:
#   1.  Integration: name ?= EVAL ...
#   2.  Integration: IF EVAL ... GOTO
#   3.  Integration: REPEAT UNTIL EVAL
#   4.  Type inference  — STR / NUM / VER / BOOL (auto-detected)
#   5.  Explicit type hints  — op:TYPE  /  :TYPE  /  : TYPE
#   6.  String operators  — EQ / eq / NE / ne / == / !=  (case-sensitive)
#   7.  Numeric operators  — == != < <= > >=
#   8.  Version operators  — == != < <= > >=
#   9.  Boolean operators  — == != with all parseBool synonyms
#   10. Lone boolean atoms — TRUE / FALSE / !TRUE / !FALSE
#   11. Compound expressions  — && / ||  with correct precedence
#   12. Macro expansion inside EVAL  — constant and variable macros
#   13. MATH as index / counter  — loop body driven by $i
#   14. MATH result fed into EVAL  — arithmetic then compare
#   15. REPEAT UNTIL EVAL  — loop termination via MATH counter
#   16. Numeric boundary conditions
#   17. Version boundary conditions
#   18. Boolean edge cases
#   19. Type inference priority  — BOOL > NUM > VER > STR
#   20. End-to-end scenario  — firmware update decision tree
#
# Reading the output:
#   Every line printed by $ok_*  should say TRUE.
#   Every line printed by $fail_* should say FALSE.
#   Section headers and narrative lines are plain text.
# =============================================================================
